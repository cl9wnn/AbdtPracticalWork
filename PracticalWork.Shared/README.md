# Репозиторий c исходным кодом общих нугет-пакетов для PracticalWork

## Наименование репозитория 

PracticalWork.Shared

## Назначение 

Cодержит исходный код модулей PracticalWork, которые собираются и распространяются как NuGet-пакеты.

Каждый модуль представляет собой отдельную библиотеку .NET, предназначенную для повторного использования в сервисах PracticalWork.

Пакеты хранятся в директории ```PracticalWork.Shared/nugetStorage```

## Пакеты
1. **PracticalWork.Shared.Abstractions** - пакет с общими абстракциями
2. **PracticalWork.Shared.Contracts** - пакет с общими контрактами для сервисов

## Требования

- [**.NET SDK: 10.0.x**](https://dotnet.microsoft.com/en-us/download/dotnet/10.0) 
- **IDE: Visual Studio 2022 (17.12+) / VS Code / Rider**

## Начало работы

1. **Проверка установленного .NET**:

    Убедитесь, что установлен необходимый .NET SDK:
     ```bash
    dotnet --version
    ```
    _Пример корректного вывода: **10.0.102**_

2.  **Восстановление зависимостей**:

    ```bash
    dotnet restore
    ```

3.  **Сборка модулей**:

    ```bash
    dotnet build -c Release
    ```

4.  **Сборка NuGet-пакетов**:

    Чтобы собрать NuGet-пакеты для всех модулей:

    ```bash
    dotnet pack -c Release
    ```
    По умолчанию после успешной сборки пакеты с расширением **.nupkg** будут находиться в директории соответствующего проекта: **src\<название модуля>/bin/Release/**

    **Пример**: ```src/PracticalWork.Shared.Abstractions/bin/Release/PracticalWork.Shared.Abstractions.1.0.0.nupkg```

    _Примечание: чтобы собрать только один nuget-пакет, необходимо выполнить те же команды, находясь в директории нужного модуля. Это соберёт только данный пакет, не трогая остальные модули._

## Использование модулей

Версии NuGet-пакетов управляются централизованно через файл **Directory.Packages.props**.

1. Добавьте ссылку на пакет в нужном .csproj используя `<PackageReference Include="<Название пакета>" />`

2. Добавьте запись в файл **Directory.Packages.props**: `<PackageReference Include="<Название пакета>" Version="<Необходимая версия>" />`

_Примечание: После публикации новой версии пакета необходимо обновить соответствующую версию в **Directory.Packages.props**, чтобы другие проекты могли использовать обновлённый модуль._
